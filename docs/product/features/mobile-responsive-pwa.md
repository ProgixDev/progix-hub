# Mobile responsiveness + installable PWA

**Status:** live · **Cross-cutting** (no single slice) · **Routes:** all
**Spec history:** specs/007-mobile-responsive-pwa (shipped 2026-06-10)

## What it does (user terms)

progixHub works cleanly on a phone — every screen reflows from ~320px up with no horizontal scrolling, the sidebar collapses behind a hamburger that opens a drawer, and the public client portal is mobile-first (clients open share links on their phones). The app is also installable: “Add to Home Screen” from a mobile browser gives a standalone, branded shortcut.

## How it works (the non-obvious 20%)

- **Breakpoint convention: Tailwind `md` (768px).** `< md` is phone/tablet, `≥ md` is the original desktop layout — every responsive class is **additive at `< md`** (e.g. `px-4 sm:px-6`), so desktop is untouched and the desktop CUJ suite is the regression guard.
- **The nav drawer is a real modal.** `AppShell` (server) delegates to a client `AppFrame` that owns the `navOpen` state. `Sidebar` renders a static `hidden md:flex` desktop aside **plus** a `md:hidden` slide-in drawer with `role="dialog"` + `aria-modal`, a focus trap, focus-move-in on open, return-focus to the hamburger on close, and a body scroll-lock. The drawer’s content is **mounted only while open** — otherwise the nav + recent-project links would be duplicated in the DOM (which broke `getByText` strict-mode in e2e and is bad for AT).
- **PWA is manifest-only (no service worker).** `src/app/manifest.ts` (`MetadataRoute.Manifest`: standalone, theme-color, brand icons) + a maskable `public/icon.svg` (from the brand `LogoMark`) + an iOS apple-touch icon generated by `next/og` (`src/app/apple-icon.tsx`, no binary asset) + a mobile-first `viewport`/`themeColor` in `layout.tsx`.
- **Gotcha — PWA assets must bypass the auth gate.** The middleware matcher excludes image extensions, but `/manifest.webmanifest` and the generated `/apple-icon` route had to be added explicitly (`src/proxy.ts`), because the OS/browser fetches them **cookie-less** when installing — otherwise they 307-redirect to sign-in and the install breaks. A signed-out e2e test guards this.
- **Reflow is class-only**: section padding `px-4 sm:px-6`; headers/rows/button-groups `flex-wrap`; `min-w-0` + `break-all`/`truncate` on long keys/URLs/file names; the documents tablist scrolls within itself; the project-detail H1 wraps (never truncates).

## Decisions & gotchas

- 2026-06-10 — Offline/service-worker, push notifications, and native packaging are **out of scope** — installability via the manifest only.
- The mobile drawer is hand-rolled (not shadcn/Radix `Sheet`); a follow-up is to promote it to a shared `Sheet` primitive so future modals inherit focus-trapping/ARIA.
- Verification runs a dedicated **`mobile` Playwright project** (Pixel 5, `e2e/mobile.spec.ts`) alongside the desktop projects; CUJs are now verified at desktop **and** mobile widths.

## CUJs covered

- No new CUJ — the existing CUJ-01..06 are re-verified at mobile width by the `mobile` Playwright project; `e2e/mobile.spec.ts` adds the no-overflow / drawer / dialog-fit / installability checks.
