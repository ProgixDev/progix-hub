# Feature Reports

Generated by `/feature-report` for every feature-track PR: a single reviewable document combining **what changed** (diff summary), **proof it works** (Playwright screenshots + verification verdicts), and **traceability** (spec acceptance criteria → evidence). This replaces "trust me" with artifacts — reviewers, PMs, and stakeholders read the report instead of reconstructing context from commits.

## Layout

```
docs/reports/
  NNN-slug.md          ← the report (committed with the PR)
  NNN-slug/img/        ← screenshots copied from artifacts/screenshots/ (committed)
  NNN-slug.pdf         ← optional, via `pnpm report:pdf NNN-slug` (not committed by default)
```

`artifacts/` itself stays gitignored — only curated evidence referenced by a report gets committed, keeping the repo lean.

## Report structure (the skill enforces this)

1. **Header** — feature, spec link, PR, date, author(+agent)
2. **What & why** — three sentences from the spec
3. **Acceptance criteria → evidence** — table: AC, test that proves it, screenshot, verdict
4. **Screenshots** — labeled, before/after when modifying existing UI
5. **Changes** — files-by-layer summary with notable decisions, generated from the real diff
6. **Verification** — `pnpm verify` + e2e output summary, persona review verdicts
7. **Follow-ups** — declined P2s, ticketed items, doc updates made

## PDF

`pnpm report:pdf <slug>` renders the markdown to PDF via Playwright's Chromium (script: `scripts/report-to-pdf.mjs`) for sharing outside the repo.
